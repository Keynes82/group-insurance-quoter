# 技术栈选型决策

## 项目约束

- 用户：约100名内部业务员
- 使用场景：内部使用，不对外
- 计算复杂度：有现成精算公式，分钟级响应可接受
- 部署：内部服务器或私有云

## 决策

| 层级 | 选择 | 理由 |
|------|------|------|
| 前端 | Streamlit | 快速原型、Python生态、适合数据输入和表格展示、无需前端开发经验 |
| 后端 | Python (FastAPI) | 精算逻辑用Python实现最自然、异步支持、API文档自动生成 |
| 数据库 | SQLite (V1) → PostgreSQL (V2) | V1用户少、无需独立数据库服务器、迁移成本低 |
| 部署 | Docker + 内部服务器 | 简单可控、无需云服务商、数据不出内网 |
| 认证 | 简单用户名密码 (V1) | 内部使用、100人规模、LDAP集成可延后 |

## 架构图

```
┌─────────────────┐
│   业务员浏览器    │
└────────┬────────┘
         │
    ┌────▼────┐
    │ Streamlit │  ← 前端界面（数据输入、结果展示）
    │  (Python) │
    └────┬────┘
         │ HTTP
    ┌────▼────┐
    │ FastAPI  │  ← 后端API（保费计算、模板管理）
    │ (Python) │
    └────┬────┘
         │
    ┌────▼────┐
    │  SQLite  │  ← 数据存储（客户信息、方案模板、历史报价）
    └─────────┘
```

## V1 技术债务

- SQLite 并发限制（100用户可能触及）
- 无缓存层（重复计算）
- 无日志和监控
- 单点部署

## V2 升级路径

- SQLite → PostgreSQL
- 添加 Redis 缓存
- 添加 Prometheus + Grafana 监控
- 负载均衡 + 多实例部署
